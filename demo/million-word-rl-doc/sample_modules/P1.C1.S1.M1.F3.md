#### 序贯决策的基本范式：算法与实践

序贯决策的核心在于将一个长程优化目标分解为逐步可执行的局部决策，并在每一步通过与环境的交互积累信息、更新策略。以下从算法流程、工程细节、典型应用和调优经验四个维度展开。

##### 通用决策循环的算法框架

无论是经典的 Q-learning 还是现代的 PPO（Proximal Policy Optimization），序贯决策算法都共享一个基本的交互-更新循环。其核心流程可描述如下：

```
算法：通用序贯决策循环
─────────────────────────────────────────────────────
输入：环境 Env，策略 π_θ，折扣因子 γ，学习率 α，最大步数 T
输出：优化后的策略参数 θ

1. 初始化参数 θ，重置环境，获得初始状态 s_0
2. for episode = 1, 2, ..., M do
3.     s ← Env.reset()
4.     for t = 0, 1, ..., T-1 do
5.         a_t ← π_θ(· | s_t)          // 依策略采样动作
6.         s_{t+1}, r_t, done ← Env.step(a_t)   // 执行动作，观测转移
7.         将 (s_t, a_t, r_t, s_{t+1}) 存入经验缓冲区 D
8.         if 满足更新条件 then
9.             从 D 中采样 mini-batch
10.            计算目标值或优势估计
11.            对 θ 执行梯度更新
12.        if done then break
13. return θ
─────────────────────────────────────────────────────
```

这一框架的关键在于第 5 步的探索-利用权衡与第 10 步的目标构造。二者共同决定了算法的收敛速度与最终性能。

以基于价值的方法为例，Q-learning 的更新目标为：

$$
Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[ r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t) \right]
$$

方括号内的量称为 **TD 误差**（Temporal Difference Error），记为 $\delta_t$。它衡量了当前估计与单步自举目标之间的差距，是序贯决策算法中最核心的信号之一。对于策略梯度方法，更新目标则转化为对期望累积回报 $J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[G_0]$ 的梯度估计，其中 $G_t = \sum_{k=0}^{T-t-1} \gamma^k r_{t+k}$ 为从时刻 $t$ 起的折扣回报。

##### 关键超参数与工程要点

**折扣因子 $\gamma$** 是序贯决策中最具全局影响力的超参数。$\gamma$ 接近 1 时，智能体倾向于规划长远，但会引入高方差；$\gamma$ 较小时，智能体目光短浅，可能陷入局部最优。实践中，稠密奖励任务常取 $\gamma \in [0.95, 0.99]$，稀疏奖励任务有时需要提升至 0.999。

**经验回放缓冲区**（Replay Buffer）的容量与采样策略直接影响离线（off-policy）算法的数据效率。缓冲区过小会导致数据相关性过强，造成训练不稳定；过大则引入过时经验，使得目标网络的估计偏差积累。DQN 的原始实现使用容量为 $10^6$ 的均匀采样缓冲区，而 PER（Prioritized Experience Replay）通过按 TD 误差加权采样显著加速了学习。

**目标网络**（Target Network）的冻结频率是另一关键工程细节。更新过于频繁会导致自举目标追逐当前估计，造成发散；更新过慢则引入偏差。Soft update 策略 $\theta^- \leftarrow \tau \theta + (1-\tau) \theta^-$（其中 $\tau \ll 1$，如 0.005）在连续控制任务中往往优于硬更新。

此外，**奖励归一化**和**观测归一化**在深度 RL 中至关重要。原始奖励信号的尺度差异会导致梯度量级不稳定，通常需要对奖励做运行时均值-方差归一化，或直接裁剪至 $[-1, 1]$。

##### 典型应用与基准表现

序贯决策框架在以下场景中已取得标志性成果：

- **Atari 游戏基准**：DQN 在 49 款 Atari 游戏上首次超越人类水平，展示了基于像素观测的端到端序贯决策能力。Rainbow 算法集成了 Double DQN、Dueling Network、PER 等六项改进，在同等样本预算下中位数得分提升约 3 倍。
- **连续控制（MuJoCo）**：SAC（Soft Actor-Critic）在 HalfCheetah、Hopper 等任务上以显著更少的环境交互次数超越 PPO，得益于其最大熵目标对探索的自动调节。
- **棋盘游戏**：AlphaZero 将序贯决策与树搜索结合，在围棋、国际象棋、将棋上均超越专业人类水平，每步决策的计算图可视为深度嵌套的序贯子问题。
- **推荐系统与广告竞价**：在工业界，以 MDP 建模用户会话、以序贯决策优化长期留存率，已成为主流范式，典型案例包括 YouTube 和 Taobao 的深度强化学习推荐系统。

##### 常见失败模式与调优经验

**过估计问题**是基于 Q 值方法的顽固病症。标准 Q-learning 因 $\max$ 操作对噪声的敏感性而系统性高估动作价值，导致策略退化。Double DQN 通过解耦动作选择与价值估计缓解了这一问题，实践中应优先采用。

**梯度爆炸与训练发散**在深度网络配合 TD 目标时频繁出现，因为目标本身依赖当前参数。标准对策包括：梯度裁剪（通常设置全局范数阈值为 0.5 或 10）、使用 Huber 损失代替 MSE，以及降低学习率。

**稀疏奖励下的探索失败**是序贯决策落地的最大工程障碍。$\varepsilon$-贪心策略在状态空间较大时几乎无效。课程学习（Curriculum Learning）、目标条件化（Goal-Conditioned RL）与内在奖励（如 RND、ICM）是当前主流的解决路径。

**样本效率**始终是现实部署的瓶颈。模型无关（model-free）方法通常需要数百万步环境交互，在真实物理系统或高成本仿真器中难以承受。此时应优先考虑基于模型（model-based）的方法（如 Dreamer、MBPO），或引入离线 RL（Offline RL）充分利用历史数据。

##### 与其他方案的实践对比与选型建议

| 场景特征 | 推荐方法 | 理由 |
|---|---|---|
| 离散动作、稠密奖励 | DQN / Rainbow | 成熟工程基础，易于调试 |
| 连续动作、样本受限 | SAC / TD3 | 样本效率高，超参数鲁棒 |
| 需要可解释性或安全约束 | 约束 MDP + 拉格朗日方法 | 支持显式安全边界 |
| 离线历史数据丰富 | IQL / TD3+BC | 避免分布外动作的悲观估计 |
| 大状态空间 + 可仿真 | Dreamer v3 | 世界模型显著压缩样本需求 |

总体原则是：**优先验证问题本身是否满足马尔可夫性**——若历史信息对最优决策不可或缺，直接用表格式或朴素深度 RL 必然失败，此时应引入递归网络（LSTM/GRU）或基于 Transformer 的记忆机制。其次，在资源受限的早期阶段应以 PPO 作为基线，其实现简单、超参数容错性强，足以快速验证任务可行性，再根据瓶颈（样本效率、探索、稳定性）针对性切换算法族。

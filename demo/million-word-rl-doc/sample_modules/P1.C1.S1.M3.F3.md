#### 奖励假设与目标：算法与实践

奖励假设（Reward Hypothesis）由 Sutton 在 2004 年正式表述：*智能体的所有目标与目的均可被表达为单一标量奖励信号的期望累积值的最大化*。这一假设是现代强化学习的基石，也是其最具争议的理论前提之一。本节聚焦于该假设在算法层面的落地方式、工程细节与实践中的典型失效模式。

---

##### 核心目标的形式化

给定折扣因子 $\gamma \in [0, 1)$，智能体的优化目标定义为折扣累积奖励的期望：

$$G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$$

策略优化的目标函数为：

$$J(\pi) = \mathbb{E}_{\pi}\left[\sum_{t=0}^{T} \gamma^t r_t \;\middle|\; s_0 \sim \rho_0\right]$$

其中 $\rho_0$ 为初始状态分布，$r_t = R(s_t, a_t, s_{t+1})$ 为环境返回的标量奖励。"奖励假设"的可操作含义在于：任何复杂的、多维度的行为目标，都必须被压缩（设计）成这一单一标量流，由此引发了奖励函数设计的一系列实践问题。

---

##### 奖励函数的设计算法流程

奖励函数设计并无统一算法，但工程中已形成若干规范化流程。以基于人类反馈的奖励学习（RLHF）为代表，其主干流程如下：

```
算法：RLHF 奖励模型训练与策略优化

输入：预训练语言模型 π_ref，人类偏好数据集 D = {(x, y_w, y_l)}
输出：对齐策略 π_θ

// Phase 1: 奖励模型训练
1. 初始化奖励模型 r_φ（通常与 π_ref 同架构）
2. for each batch (x, y_w, y_l) in D:
     L_RM(φ) = -E[log σ(r_φ(x, y_w) - r_φ(x, y_l))]
     更新 φ ← φ - α∇L_RM(φ)

// Phase 2: 策略优化（PPO + KL 惩罚）
3. 初始化策略 π_θ ← π_ref
4. for each training step:
     a. 采样 (x, y) ~ π_θ
     b. 计算奖励：r̂(x,y) = r_φ(x,y) - β · KL[π_θ(·|x) ‖ π_ref(·|x)]
     c. 用 PPO 目标更新 π_θ

// KL 惩罚项防止策略过度偏离参考模型（奖励黑客）
```

在更基础的稀疏奖励场景中，Hindsight Experience Replay（HER）通过事后目标重标记将失败轨迹转化为正样本：

```
算法：HER（事后经验回放）

1. 执行 episode，收集轨迹 τ = {(s_t, a_t, r_t, s_{t+1})}
2. 将原始 (s, a, r, s') 存入回放缓冲区
3. 从同一 episode 中采样 k 个"事后目标" g' ∈ {s_{t+1}, ..., s_T}
4. 对每个 g'，重新计算奖励 r' = R(s, a, g') 并存入缓冲区
5. 用增广缓冲区训练 DQN / SAC 等算法
```

##### 关键超参数与工程要点

**折扣因子 $\gamma$** 是奖励设计中影响最深远的超参数。$\gamma \to 1$ 时智能体具有长期视野但方差激增；$\gamma \to 0$ 时行为近视。实践中，连续控制任务常取 $\gamma = 0.99$，而回合制对弈任务可取 $\gamma = 1.0$（有限步数保证收敛）。

**奖励缩放（Reward Scaling）** 对策略梯度算法至关重要。若奖励量级与值函数估计不匹配，梯度的信噪比会急剧恶化。常见做法是对奖励做运行均值归一化：

$$\hat{r}_t = \frac{r_t - \mu_r}{\sigma_r + \epsilon}$$

**奖励裁剪（Reward Clipping）** 被 DQN 系列广泛采用，将所有奖励截断至 $[-1, 1]$，牺牲奖励幅度信息换取跨游戏的通用性。

**奖励塑形（Reward Shaping）的势函数约束**：为了在保证最优策略不变的前提下加速学习，Ng et al. 证明，只有基于势函数的塑形形式

$$r'(s, a, s') = r(s, a, s') + \gamma \Phi(s') - \Phi(s)$$

才能保证策略不变性（potential-based shaping invariance），其中 $\Phi: \mathcal{S} \to \mathbb{R}$ 为任意势函数。违反此约束是奖励工程中最常见的隐性错误。

##### 典型应用与基准表现

| 场景 | 奖励设计策略 | 代表工作 | 关键结论 |
|------|------------|---------|---------|
| Atari 游戏 | 稀疏 + 裁剪至 $[-1,1]$ | DQN (Mnih 2015) | 裁剪使 57 款游戏统一训练成为可能 |
| 机器人操控 | 稀疏 + HER | OpenAI Robotics (2018) | HER 将抓取成功率从 <5% 提升至 >90% |
| LLM 对齐 | 人类偏好 + KL 惩罚 | InstructGPT (2022) | 1.3B RLHF 模型偏好度超过 175B SFT 模型 |
| 棋盘游戏 | 终局稀疏奖励 | AlphaZero (2017) | 纯稀疏奖励 + MCTS 无需中间奖励工程 |

##### 常见失败模式与调优经验

**奖励黑客（Reward Hacking）** 是奖励假设在实践中遭遇的最根本挑战。代理奖励（proxy reward）是对真实目标的近似，智能体往往找到代理奖励的高分解法，却与真实目标背道而驰。经典案例包括：赛艇游戏中智能体通过原地打转持续触碰加速圈得分，而非完成比赛。缓解方法包括增加多样性约束、定期人工审查轨迹、以及使用 AI 反馈（RLAIF）替代纯代理奖励。

**奖励稀疏导致的探索失败** 在高维状态空间中极为普遍。HER 与基于计数的探索（Count-Based Exploration）是两条主要技术路线，前者适合目标导向任务，后者适合无结构探索场景。

**折扣因子与任务时间尺度不匹配** 会导致智能体忽视长时延奖励。诊断方法是绘制 Q 值与蒙特卡洛回报的相关性：若二者严重偏离，往往是 $\gamma$ 设置过低或奖励量级失配。

**多目标混入单一标量的权重敏感性**：当奖励为多目标加权和 $r = \sum_i w_i r_i$ 时，权重的微小变化可导致最终策略行为的剧烈改变，且最优权重通常无法通过直觉确定，需要借助多目标 RL（MORL）或帕累托前沿分析。

##### 与其他方案的对比与选型建议

相较于**逆强化学习（IRL）**，显式奖励设计工程成本更低、训练更稳定，适合目标明确的任务；IRL 在奖励难以手工定义但专家演示易于获取时更有优势。相较于**模仿学习（BC/GAIL）**，基于奖励的 RL 具备超越演示的上限潜力，但对奖励设计质量高度敏感。

实际选型时建议遵循以下优先级：若任务存在天然密集奖励（如物理仿真中的位移量），直接使用；若奖励稀疏但目标结构清晰，优先使用 HER；若目标难以形式化，优先收集人类偏好数据走 RLHF 路线；仅当以上均不适用时，才考虑从专家演示中学习奖励（IRL）。任何情况下，都应将奖励函数视为一级代码进行版本管理与测试，而非事后拼凑的调参旋钮。

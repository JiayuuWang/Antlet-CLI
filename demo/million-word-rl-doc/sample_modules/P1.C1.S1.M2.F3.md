#### 智能体-环境交互回路：算法与实践

智能体与环境的交互回路是强化学习系统的运行骨架。无论上层算法如何演进——从表格型 Q-learning 到深度策略梯度，再到大模型驱动的 Agentic RL——底层的交互协议始终保持相同的抽象形式。理解并正确实现这一回路，是构建可靠 RL 系统的第一步。

##### 标准交互回路的算法描述

下面给出单智能体、离散时间交互回路的通用伪代码，它直接对应马尔可夫决策过程 $(S, A, P, R, \gamma)$ 的操作语义。

```
Algorithm: Agent-Environment Interaction Loop
─────────────────────────────────────────────
输入：策略 π，环境 env，最大步数 T，折扣因子 γ
输出：轨迹 τ = {(s_t, a_t, r_t, s_{t+1})}，累积回报 G

1:  初始化回放缓冲区 B ← ∅
2:  s_0 ← env.reset()
3:  G ← 0,  t ← 0
4:  while t < T and not env.is_terminal() do
5:      a_t ← π(s_t)                      // 动作选择
6:      s_{t+1}, r_t, done, info ← env.step(a_t)
7:      B.push(s_t, a_t, r_t, s_{t+1}, done)
8:      G ← G + γ^t · r_t                 // 在线累积折扣回报
9:      if 满足更新条件 then
10:         从 B 采样 mini-batch → 执行策略更新
11:     s_t ← s_{t+1}
12:     t ← t + 1
13: return τ, G
```

上述回路的核心数学对象是**折扣累积回报**

$$G_t = \sum_{k=0}^{T-t-1} \gamma^k r_{t+k+1}$$

其中 $\gamma \in [0, 1)$ 对远期奖励施加指数衰减，直接决定智能体的"远视"程度。当 $\gamma \to 1$ 时，智能体等权对待所有未来奖励，适用于有限长 episode；当 $\gamma$ 较小时，智能体更关注近期收益，有助于在稀疏奖励环境中加速早期学习，但可能导致短视策略。

##### 关键实现细节与超参数

**环境封装（Wrapper）层**是工程实践中最容易被忽视的环节。原始环境的观测往往需要经过归一化、帧堆叠、奖励裁剪等预处理，才能送入神经网络。以 Atari 游戏为例，标准做法包括：将 RGB 帧灰度化并缩放到 $84 \times 84$，堆叠连续 4 帧作为状态 $s_t \in \mathbb{R}^{84 \times 84 \times 4}$，并将奖励裁剪至 $\{-1, 0, +1\}$。这些操作虽然简单，却对最终性能有决定性影响——省略帧堆叠会使状态不满足马尔可夫性，导致策略无法感知速度信息。

**步长与 episode 边界处理**需要特别注意 `done` 信号的语义区分。现代环境（如 Gymnasium）将 `terminated`（到达终止状态）与 `truncated`（到达最大步数被强制截断）分开返回。在计算目标值时，两者处理方式不同：

- `terminated = True`：自举目标为 $r_t$，不加未来价值项
- `truncated = True`：自举目标应为 $r_t + \gamma V(s_{t+1})$，否则会系统性低估 episode 被截断时的价值

混淆两者是 off-policy 算法（如 DQN、SAC）中值函数低估的常见来源之一。

**回放缓冲区容量**是离线采样算法的核心超参数。DQN 原始论文使用 $10^6$ 步容量，但对内存受限场景，$5 \times 10^4$ 到 $2 \times 10^5$ 通常已足够，同时需要相应调整预热步数（warm-up steps）。批大小（batch size）$B$ 通常取 32 到 256，过大会降低数据效率，过小则导致梯度估计方差过高。

**动作选择策略**在探索-利用权衡中至关重要。$\varepsilon$-greedy 的衰减调度直接影响样本效率：

$$\varepsilon_t = \max\left(\varepsilon_{\min},\ \varepsilon_0 \cdot \exp\left(-\frac{t}{\tau_{\varepsilon}}\right)\right)$$

过快衰减会导致智能体在价值函数尚未收敛时便陷入局部最优；过慢衰减则浪费大量样本在随机探索上。

##### 典型应用与基准表现

在 Atari-57 基准上，标准 DQN 交互回路（含上述环境封装）在 200M 帧训练后可在约半数游戏中超越人类水平，而去掉帧堆叠或奖励裁剪后，中位数得分会下降 30\%–50\%。Rainbow DQN 在相同回路框架下整合了 6 项改进，以 44M 帧达到了 DQN 200M 帧的等效性能，说明交互回路本身的正确性是算法改进发挥作用的前提。

在连续控制领域（MuJoCo Humanoid-v4），SAC 采用相同的交互回路结构，以 $3 \times 10^6$ 步训练达到平均回报 6000+，而 PPO 需要 $10^7$ 步才能达到同等水平，体现了 off-policy 回路在样本效率上的优势。

##### 常见失败模式与调优经验

**奖励尺度失配**是最常见的隐性问题。若不同任务分支的奖励量级相差 $10^2$ 以上，神经网络的梯度将被大奖励来源主导，小奖励信号被淹没。解决方案是对奖励进行运行时归一化（如 PopArt），或使用奖励裁剪配合价值函数的尺度适配。

**观测不一致**：在向量化环境（多进程并行）中，若各子环境的随机种子未正确设置，会导致数据分布偏斜。建议显式传入不同种子并在日志中记录，便于复现。

**梯度爆炸**在交互回路与网络更新耦合过紧时容易出现。标准做法是对梯度施加全局范数裁剪（clip by global norm，阈值通常取 0.5 到 10），并在回路中监控 TD 误差的滑动均值，若均值持续上升则提前终止并重置学习率。

##### 与其他方案的对比与选型建议

| 交互模式 | 代表算法 | 样本效率 | 实现复杂度 | 适用场景 |
|---|---|---|---|---|
| On-policy 单步 | A2C | 低 | 低 | 快速验证、简单任务 |
| On-policy 多步 | PPO | 中 | 中 | 连续控制、稳定性优先 |
| Off-policy 回放 | DQN / SAC | 高 | 中高 | 样本昂贵、离散/连续动作 |
| 异步并行 | IMPALA / Ape-X | 很高 | 高 | 大规模分布式训练 |

对于大多数研究场景，**PPO + 向量化环境**（4–32 个并行 worker）是起点首选：实现简单、调参容忍度高、复现性强。当样本成本显著（如真实机器人、仿真代价高）时，应切换至 **SAC 或 TD3** 的 off-policy 回路，以获得 3–10 倍的样本效率提升。进入生产级分布式训练阶段后，IMPALA 的 actor-learner 解耦架构可将吞吐量提升至单机方案的数十倍，但需要仔细处理 off-policy 修正（V-trace）以保证收敛性。

无论选择哪种方案，交互回路的正确性验证应先于算法调优：用随机策略跑通完整回路并检查奖励统计，确认 episode 边界处理无误，再接入学习模块，可避免大量难以定位的训练失败。
